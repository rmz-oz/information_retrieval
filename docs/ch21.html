21.1 Viewing the Web as a Graph

The web graph is a massive directed graph in which every web page is a node and every hyperlink is a directed edge.

Two Core Intuitions

1. Anchor Text

The anchor text of a link pointing to page B is a good description of page B. Users create anchor text when they describe a page "in their own words."

2. Link = Endorsement

A link from page A to page B means that A has "endorsed" B. (Internal links — coming from a shared template — are excluded from this assessment.)

21.1.1 Anchor Text and the Web Graph

Classic example: IBM's homepage didn't contain the word computer, and Yahoo's page didn't contain portal. Yet the anchor texts of links to these pages did contain these words. As a result:

Power of Anchor Text Anchor text can describe the target page better than the target page's own content. Search engines use anchor text as an indexing term for the page it links to.
Link Spam Risk Derogatory anchor text examples such as "Big Blue" → IBM or "evil empire" can lead to deliberately crafted misleading anchor text used to manipulate search engines (Google bombing). This is a form of spam.

Extended anchor text: The text window surrounding a link can also contain information as useful as the anchor text itself.

21.2 PageRank

PageRank assigns a numerical score between 0 and 1 to every web page. This score is a static quality measure determined by the page's link structure — it is independent of any query.

Random Surfer Model

A random surfer starting on a web page selects one of the page's outlinks uniformly at random and clicks it at each step. As in Figure 21.1: if node A has 3 outlinks (B, C, D), each is followed with probability 1/3.

Teleport Operation What happens at a page with no outlinks (dangling node)? Or if the surfer gets stuck? Solution: teleport. At each step, the surfer teleports to any page with probability α (one of N pages, each with equal probability 1/N). Typical α = 0.1.

When Outlinks Exist

Follow a random outlink with probability (1 − α)
Teleport to any page with probability α

When No Outlinks Exist

Always teleport
(to any web page with equal probability)

21.2.1 Markov Chains

The random surfer model is formulated as a Markov chain: each web page is a state, and transition probabilities are determined by the link structure between pages.

∀i, j, P_ij ∈ [0,1] and ∀i, Σⱼ P_ij = 1

The transition probability matrix P is obtained as follows:

  1. In the adjacency matrix A, replace any row containing no 1s with all elements set to 1/N.
  2. Divide each 1 in a row by the number of 1s in that row (equal outlink distribution).
  3. Multiply the resulting matrix by (1 − α).
  4. Add α/N to each element → this is matrix P.
Stochastic Matrix Matrix P is a stochastic matrix: all elements are in [0,1] and each row sums to 1. It has a principal left eigenvector corresponding to its largest eigenvalue, which is 1.

Ergodic Markov Chain

A Markov chain is ergodic if the positive transition probability between any two states eventually becomes positive as the chain runs. Requirements:

Irreducibility

Every state is reachable from every other state with non-zero probability.

Aperiodicity

States do not split into periodic subsets; transitions do not follow fixed cyclic patterns.
Theorem 21.1 (Steady-State) For every ergodic Markov chain, where η(i,t) = total visits to state i in t steps:
lim (t→∞) η(i,t)/t = π(i)
This π(i) value is the steady-state probability of state i and is the page's PageRank.

21.2.2 PageRank Computation: Power Iteration

PageRank values are found as the principal left eigenvector of matrix P:

π̃ · P = 1 · π̃

Power Iteration method:

  1. Start with x̃₀ (e.g. (1, 0, 0, ...) — surfer starts at first page)
  2. x̃₁ = x̃₀ · P
  3. x̃₂ = x̃₁ · P = x̃₀ · P²
  4. ... as t grows, x̃ₜ converges to the steady-state distribution
  5. Stop when the change between successive iterations falls below a threshold
PageRank is Query-Independent PageRank values are precomputed and used only as one factor at query time. Since the relative ranking of pages can vary by query, PageRank alone is insufficient for ranking — it is combined with cosine similarity and other factors.

21.2.3 Topic-Specific PageRank

In standard PageRank, teleport is distributed equally across the entire web. In Topic-Specific PageRank, teleport goes only to a set S of pages on a specific topic (e.g. sports).

Sports PageRank (π̃ₛ)

Teleport only to sports pages. Sports pages receive higher scores.

Politics PageRank (π̃ₚ)

Teleport only to politics pages. Politics pages are favored.

Personalized PageRank: If the user is 60% interested in sports and 40% in politics:

Personalized PageRank = 0.6 · π̃ₛ + 0.4 · π̃ₚ
Efficiency There is no need to compute a separate matrix for each user profile. A linear combination of topic-based PageRank vectors is sufficient — this follows from the linear structure of Markov chains.

21.3 Hubs and Authorities (HITS)

HITS (Hyperlink-Induced Topic Search), unlike PageRank, is a query-sensitive link analysis method. Each page is assigned two separate scores:

Hub Score h(v)

Good hubs link to many good authorities. "Guiding" pages — like manually compiled resource lists.

Authority Score a(v)

Good authorities are pointed to by many good hubs. "Expert" pages — primary sources on a topic.
Circular Definition — Iterative Solution Good hubs → point to good authorities
Good authorities → are pointed to by good hubs
This circular definition is resolved by iterative computation.

Iterative Update Rules

h(v) ← Σ a(y) [for all y with v → y]
a(v) ← Σ h(y) [for all y with y → v]

In matrix form (A = adjacency matrix):

h⃗ ← A · a⃗
a⃗ ← Aᵀ · h⃗

Combined:

h⃗ ← AAᵀ · h⃗
a⃗ ← AᵀA · a⃗
Eigenvalue Connection These equations resemble the eigenvector equations of matrices AAᵀ and AᵀA. The hub and authority vectors are the principal eigenvectors of AAᵀ and AᵀA respectively.

HITS Algorithm — Summary

  1. Collect a subset of target web pages (base set)
  2. Build the graph over this subset, compute matrices AAᵀ and AᵀA
  3. Find principal eigenvectors → hub and authority scores
  4. Return the highest-scoring hubs and authorities

21.3.1 Selecting the Web Subset

Why don't we use direct text query pages? Because good authority pages may not contain the query term (e.g. the word "computer" does not appear on IBM's site, yet IBM is a computer company).

Root Set

~200 pages matching the text query. These form the core of the base set.

Base Set

Root set + pages linking to the root set + pages linked from the root set. Both hubs and authorities are included here.
Multilingual Side Effect When HITS runs, pages outside the query language can emerge due to base set expansion. E.g. the English query "japan elementary schools" found Japanese pages as authorities — this stems from pure link structure, not translation.

Question 1 — PageRank Power Iteration Computation ▼

Question: For the 3-node Markov chain in Figure 21.2 (A→B 0.5, A→C 0.5; B→A 1; C→A 1), find the transition probability matrix and compute the steady-state distribution with teleport rate α = 0.5.

Answer:

Transition matrix (before teleport):

P_raw = | 0    0.5  0.5 |
        | 1    0    0   |
        | 1    0    0   |

With teleport added at α = 0.5: P = (1−0.5)·P_raw + (0.5/3)·J (J: all-ones matrix)

P = | 1/6   2/3   1/6 |
    | 5/12  1/6   5/12|
    | 1/6   2/3   1/6 |

By symmetry: states 1 and 3 share the same steady-state probability (p). For state 2: 1−2p. Solving π̃ = π̃·P: p = 5/18, therefore π̃ = (5/18, 4/9, 5/18).

Question 2 — HITS Hub and Authority Update ▼

Question: If all hub and authority scores are initialized to 1, what is the authority score of a node with in-degree 3 after one iteration? What is the hub score of a node with out-degree 2?

Answer:

Authority update rule: a(v) ← Σ h(y) (for all y such that y → v)

Since all h(y) = 1 initially, for v with in-degree 3: a(v) = 1 + 1 + 1 = 3

Hub update rule: h(v) ← Σ a(y) (for all y such that v → y)

Since all a(y) = 1 initially, for v with out-degree 2: h(v) = 1 + 1 = 2

General rule: after the first iteration, a(v) = in-degree(v), h(v) = out-degree(v).

Question 3 — Ergodic Markov Chain Conditions ▼

Question: Consider a three-state Markov chain: probability 1 from A to B; pA from B to A, (1−pA) to C; probability 1 from C to A. For which values pA ∈ [0,1] is the chain ergodic?

Answer:

Irreducibility: If the path A→B→C→A exists, all states are reachable from each other. To reach C from B, (1−pA) > 0, so pA < 1.

Aperiodicity: The A→B→A cycle could create period 2. However, if B can also go to C (pA < 1) and C goes to A, cycles of different lengths exist, ensuring aperiodicity.

If pA = 1, C is unreachable from B → not irreducible.

Conclusion: The chain is ergodic for 0 ≤ pA < 1.

Question 4 — Topic-Specific PageRank and Personalization ▼

Question: Why do we not need to compute a separate PageRank vector for each user profile? How are topic-based vectors used?

Answer:

Thanks to the linear structure of Markov chains, a user's personalized PageRank vector can be expressed as a linear combination of topic-based PageRank vectors.

User interest profile: proportion q sports, (1−q) politics:

π̃_personal = q · π̃_sports + (1−q) · π̃_politics

This means computing PageRank once per topic is sufficient. For different user profiles, these precomputed vectors simply need to be combined with different weights — no new matrix computation is needed.

Question 5 — HITS and PageRank Spam Defense ▼

Question: What is "link spam" and how do PageRank and HITS handle it?

Answer:

Link Spam: An attacker creates many fake pages to artificially inflate the in-link count of a target page. The goal is to boost the target page's quality score.

PageRank's defense:

  • The teleport operation keeps the chain ergodic and limits the accumulation of links from a single source.
  • Since links from high-PageRank pages carry much more weight, contributions from low-quality spam pages are already small.
  • Search engines detect spam using additional signals (domain trust, link pattern analysis).

HITS's weakness:

  • Since HITS is computed at query time, spam pages can be included during the base set construction stage.
  • "Topic drift": the base set can drift toward unrelated topics, making it more susceptible to manipulation.
