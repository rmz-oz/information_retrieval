6.1 Parametric and Zone Indexes

Chapter 6 moves beyond Boolean retrieval to ranked retrieval: assigning a numeric score to each document and returning the top-k results. Documents contain two types of structured content:

Fields (Parametric Index)

Short metadata: author, date, language. Supports equality queries (e.g., find all docs by a given author). Stored in a parametric index.

Zones (Zone Index)

Arbitrary text regions: title, abstract, body. Each zone gets its own postings list, enabling zone-specific Boolean queries. Stored in a zone index.

Weighted Zone Scoring

Assign a weight gi to each zone i (with Σgi = 1). Each zone contributes a Boolean score si ∈ {0, 1} (does the doc match the query in that zone?). The overall document score is:

score(d) = Σi gi · si

This is ranked Boolean retrieval, implemented by the ZONESCORE algorithm which walks through postings for the query term across all zones and accumulates weighted scores.

ZONESCORE Algorithm For each query term t, retrieve postings for (t, zone_i) for every zone i. For each document in those postings, accumulate gi · si. Return documents sorted by descending score.
Key constraint on zone weights: Different Boolean match functions can be used per zone (e.g., exact phrase match in title, stemmed match in body). The only requirement is si ∈ {0,1} and Σgi = 1.

Machine-Learned Relevance Weights

Instead of setting zone weights manually, learn them from training examples — (query, document, relevance) triples. For a two-zone system (title T, body B), the optimal weight g* that minimizes ranking errors is:

g* = (n10r + n01n) / (n10r + n01n + n10n + n01r)
NotationsTsBRelevant?Meaning
n10r10YesTitle matched, doc relevant → title is a good signal
n01n01NoBody matched, doc non-relevant → body is noisy
n10n10NoTitle matched, doc non-relevant → title false positive
n01r01YesBody matched, doc relevant → body missed by title
Why sT = sB cases are excluded: When both zones give the same score, score(d) = g·s + (1−g)·s = s regardless of g. These examples carry no information about the optimal g and are correctly excluded.

6.2 Term Frequency and Weighting

The bag-of-words model ignores word order and represents a document by its term frequencies. tft,d is the number of times term t appears in document d. High tf suggests relevance, but raw tf is not ideal — a document mentioning 'car' 10× is not 10× more relevant than one mentioning it once.

Inverse Document Frequency (IDF)

Rare terms are more informative than common ones. The IDF of term t is:

idft = log(N / dft)

where N = total documents and dft = documents containing t.

idf = 0

Term appears in every document (df = N). Contributes nothing to ranking — same effect as stop word removal.

idf = log N

Term appears in exactly one document (df = 1). Maximum discrimination.

Always ≥ 0

df ≤ N → N/df ≥ 1 → log(N/df) ≥ 0. IDF is always non-negative and finite.
Log base & ranking: Changing the log base multiplies all idf values by a constant factor, so the relative ranking of documents is unchanged. Base 10, natural log, and base 2 are all equivalent for ranking. Base 2 gives an information-theoretic interpretation: idf = bits needed to identify which documents contain the term.

TF-IDF Weighting

Combining tf and idf gives the tf-idf weight — highest for terms that appear many times in a document but are rare across the collection:

tf-idft,d = tft,d × idft

The score of query q against document d sums tf-idf weights over matching terms:

Score(q, d) = Σt ∈ q ∩ d tf-idft,d
Is tf-idf bounded by 1? No. Raw tf-idf is unbounded — tf=50 and idf=5 gives tf-idf=250. Only after cosine normalization are the vector components in [0, 1].

6.3 The Vector Space Model

Each document d is represented as a vector V(d) ∈ ℝ|V|, where |V| is the vocabulary size and each dimension corresponds to a term. The weight of term t in document d is its tf-idf score. Queries are represented as vectors in the same space.

Cosine Similarity

Similarity is measured by the cosine of the angle between vectors:

sim(d1, d2) = V(d1) · V(d2) / (|V(d1)| × |V(d2)|)

The denominator length-normalizes both vectors by their Euclidean lengths: |V(d)| = √(Σ wi2). After normalization, each vector lies on the unit hypersphere and similarity is just a dot product.

Why Length Normalization?

Without it, longer documents accumulate larger dot products simply by containing more terms. Normalization ensures similarity reflects topical relevance, not document length.

Unit Vector Property

After normalization, Σ vi2 = 1 (sum of squares = 1). Cosine similarity ranges from −1 to 1; for tf-idf (non-negative weights), it ranges from 0 to 1.

Queries as Vectors

A query q is treated as a short document and embedded in the same vector space. The score of a document d for query q is: score(q, d) = v(q) · v(d) (dot product of unit vectors).

COSINESCORE Algorithm

COSINESCORE(q):
  A[d] = 0 for all d            // accumulator array, O(N)
  for each term t in q:
    fetch postings list for t
    for each (d, tf_{t,d}) in postings:
      A[d] += tf-idf_{t,d}      // accumulate, O(|q| · L_ave)
  for each d with A[d] > 0:
    A[d] /= |V(d)|              // cosine normalize
  return top-k docs by A[d]
Complexity: O(N + |q| · Lave) Initialization: O(N). Postings traversal: O(Σt∈q Lt) ≈ O(|q| · Lave). Final normalization: O(N). Term-at-a-time strategy — contrast with document-at-a-time processing.

Cosine ↔ Euclidean Distance Equivalence

For unit vectors (|q| = |d| = 1):

|q − d|2 = |q|2 − 2q·d + |d|2 = 2(1 − cosθ)

Since √(2(1−x)) is strictly decreasing, minimizing Euclidean distance = maximizing cosine similarity for unit vectors.

6.4 Variant tf-idf Functions

Sublinear tf Scaling

Raw tf grows linearly — too aggressive. Sublinear (log) tf dampens this growth:

wft,d = 1 + log(tft,d)   if tft,d > 0,   else 0

Doubling tf does not double relevance — the log function models diminishing returns for repeated occurrences.

Maximum tf Normalization

Normalize tf by the maximum tf in the document to reduce the effect of unusually long documents:

ntft,d = a + (1 − a) × tft,d / tfmax(d)

Smoothing constant a ≈ 0.4 prevents very low weights for less-frequent terms.

SMART Notation (ddd.qqq)

SMART notation compactly encodes a tf-idf scheme as ddd.qqq — three characters for document weighting, three for query weighting. Each character specifies: (1) tf method, (2) idf method, (3) normalization.

Pivoted Document Length Normalization

Standard cosine normalization over-penalizes long documents and helps short ones. Empirically, there is a pivot length p where the true relevance probability intersects the cosine curve. Pivoted normalization rotates around this pivot:

Normalizer = a · |V(d)| + (1 − a) · piv

A practical approximation uses unique terms ud: a · ud + (1 − a) · piv. With slope a < 1, the normalizer is smaller than Euclidean length for short documents (boosting them) and larger for long documents (less penalty than pure cosine). This corrects the bias.

Question 1
In weighted zone scoring, what is the formula for a document's score given zone weights gi and Boolean zone scores si?
score(d) = Σ gi · si with constraints gi ≥ 0, Σgi = 1, and si ∈ {0, 1}. This is a weighted sum of Boolean zone matches — zones with higher gi contribute more to the final score.
Question 2
For a two-zone system with training data giving n10r=3, n01n=2, n10n=1, n01r=1, what is the optimal title zone weight g*?
g* = (n10r + n01n) / (n10r + n01n + n10n + n01r) = (3+2)/(3+2+1+1) = 5/7 ≈ 0.714. The high value reflects that title matches are more reliable indicators of relevance in this training data.
Question 3
Why are training examples where sT = sB excluded from the optimal zone weight formula?
When both zones give the same Boolean score s, the weighted zone score simplifies to s no matter what g is. Changing g cannot improve or worsen the ranking for these examples, so they are irrelevant to the optimization and correctly excluded.
Question 4
What is the IDF of a term that appears in every document in the collection?
If dft = N (appears in every document), then idft = log(N/N) = log(1) = 0. Multiplied by any tf, tf-idf = 0 — the term contributes nothing to ranking. This is the same effect as automatic stop word removal.
Question 5
How does changing the logarithm base (e.g., from base 10 to natural log) affect document ranking with tf-idf?
logb2(N/df) = logb1(N/df) / logb1(b2). All idf values are scaled by the same constant factor. Since ranking is based on relative ordering of scores, the ranking is identical regardless of base.
Question 6
In the vector space model, what does Euclidean length normalization accomplish?
Without normalization, longer documents accumulate larger dot products simply by containing more terms. Dividing by |V(d)| = √(Σ wi2) converts vectors to unit vectors so that similarity measures direction (topic), not magnitude (length).
Question 7
What is the time complexity of the COSINESCORE algorithm?
Accumulator initialization: O(N). Postings traversal for |q| terms of average length Lave: O(|q| · Lave). Final normalization sweep: O(N). Total: O(N + |q| · Lave).
Question 8
For unit vectors, what is the relationship between cosine similarity and Euclidean distance?
|q−d|² = |q|²−2q·d+|d|² = 1−2cosθ+1 = 2(1−cosθ). Since √(2(1−x)) is strictly decreasing, minimizing |q−d| is equivalent to maximizing cosθ. Both rankings are identical for unit vectors.
Question 9
What does sublinear tf scaling wft,d = 1 + log(tft,d) address?
Raw tf grows linearly, but relevance exhibits diminishing returns — a document mentioning 'car' 10 times is not 10× more relevant than one mentioning it once. The log function grows sublinearly (e.g., log(10) ≈ 2.3, not 10), better modeling this relationship.
Question 10
In SMART notation, what does the scheme 'lnc.ltc' mean?
SMART is ddd.qqq. 'lnc': l=log tf, n=no idf, c=cosine normalization (for documents). 'ltc': l=log tf, t=standard idf, c=cosine normalization (for query). A common scheme that applies idf only on the query side.
Question 11
What is the augmented tf formula (SMART code 'a') and what problem does it solve?
Augmented tf: ntf = a + (1−a) × tf/tfmax(d), with a ≈ 0.4 (smoothing constant). It normalizes tf relative to the most frequent term in the document, reducing the effect of unusually long documents while the smoothing constant prevents very low weights for rare terms.
Question 12
What problem does pivoted document length normalization solve that standard cosine normalization does not?
Empirically, there is a pivot document length p where the true relevance probability intersects the cosine normalization curve. Pivoted normalization uses Normalizer = a·|V(d)| + (1−a)·piv (a < 1), which is smaller than Euclidean length for short docs (boosting them) and larger for long docs (less penalty).
Question 13
After stemming 'jealous' and 'jealousy' to the same stem, what happens to the combined term's idf compared to each individual term's idf?
The new df = |{d : 'jealous'∈d} ∪ {d : 'jealousy'∈d}| ≥ max(df('jealous'), df('jealousy')). Since df increases, idf = log(N/df) decreases. However, tf also increases (adding both forms' frequencies), so the net effect on tf-idf depends on which change dominates.
Question 14
In the ntc.atc scoring scheme, what do the document and query sides use?
ntc = natural tf (n), standard idf (t), cosine normalization (c) for documents. atc = augmented tf (a), standard idf (t), cosine normalization (c) for queries. This balanced scheme is considered good in practice — it applies tf-idf to documents and smoothed augmented tf-idf to queries.
Question 15
How are out-of-vocabulary (OOV) query terms handled in the standard vector space model?
The standard approach is to ignore OOV terms — the query vector has zero components for terms not in the index. For an OOV term with df=0, even if we assign a high idf, no document contains it (dt=0 for all d), so its contribution to scores is 0 regardless. The remaining terms determine the ranking.

All 8 combinations produce distinct scores (no two subsets of {0.20, 0.31, 0.49} have the same sum) → 8 distinct score values.

5. Pivoted Normalization vs. Cosine & OOV Terms ›

Question: (a) Explain why standard cosine normalization has a bias against long documents, and how pivoted normalization corrects it. (b) How are out-of-vocabulary terms handled in 'ntc.atc' scoring for query 'coyote insurance', where 'coyote' never appears in the collection?

Part (a) — Cosine bias and pivoted correction:

The bias: Standard cosine normalization divides by |V(d)| = √(Σwi2), the Euclidean length. Empirically, longer documents tend to have larger Euclidean lengths, so they are penalized more heavily. This causes a short-document bias — short documents get artificially boosted even when long documents are genuinely more relevant.

The pivot: Plotting the probability of relevance against document length, there is a pivot point p where the cosine normalization curve intersects the true relevance curve. Documents shorter than p are over-rewarded by cosine; longer documents are over-penalized.

Pivoted normalization uses: Normalizer = a · ud + (1−a) · piv, where ud = unique terms in d, a < 1, piv = pivot value (average unique terms across the collection). This "rotates" the normalization curve around the pivot:

  • For short documents (ud < piv): normalizer < Euclidean length → score boosted vs. pure cosine
  • For long documents (ud > piv): normalizer > piv but < Euclidean length → less penalized vs. pure cosine

Part (b) — OOV term 'coyote' in 'ntc.atc':

'Coyote' has df=0 → idf = log(N/0) is undefined. Standard approach: drop OOV terms.

Query becomes effectively 'insurance' only. Under 'atc': augmented tf = 1.0 (tf=1, tf_max=1), idf(insurance) = log(N/df_insurance) ≈ 1.72. Normalized query weight = 1.0×1.72/1.72 = 1.0 (trivially).

For document weights (ntc): vinsurance,d = tf(insurance,d) × idf(insurance) / |V(d)|. The ranking reduces to ranking documents by their cosine-normalized insurance tf-idf weight.

Why does 'coyote' contribute nothing? Even if we assign a large idf to 'coyote', every document has tf(coyote,d)=0, so tf-idf(coyote,d)=0 for all d. The 'coyote' component of the dot product is qcoyote×0=0 regardless of the query weight assigned.
